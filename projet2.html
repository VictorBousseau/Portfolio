    Création d’un outil décisionnelle sur les prix de l’immobilier en Bretagne et en France

    1 - Contexte et objectifs

    Dans le cadre du cours d’analyse et conception d’un outil décisionnelle nous devions mettre en place un système décisionnel permettant à des acteurs de la filière (agents immobiliers, des promoteurs immobiliers des responsables de l’urbanisme) d’analyser et de comprendre la situation et l’évolution du marché de l’immobilier et les enjeux économiques et sociétaux.

    Le but de ce projet était de crée modèle relationnel permettant le calcul des indicateurs de supervision en fonction des axes d’analyses en optimisant les performances et le stockage, et d’intégrer différentes sources de données de manière automatique.

    2 - Présentations des données

    Pour ce projet nous avons accès a des données de la CEREMA, qui sont des données liées aux ventes immobilière. Les données sont à disposition sous forme d’un fichier Excel par département. Nous retrouvons donc des informations de localisation, de prix de vendre, de valeur foncière dans ces données. En parallèle de ces données nous devions les mettre en relations avec des indicateurs sociaux économique. Pour cela nous avons trouvée des données sur le site de l’INSEE ce qui nous a permis d’avoir pour chaque « code INSEE » des indicateurs sur les professions, des informations sur les habitants leurs tranche d’âges ou sur des catégories sociales professionnelles.

    Après avoir cherché et sélectionnée nous données nous devions donc les liée, les nettoyer et crée de nouvelles variables.

    3 - Création des processus et script

    Pour la création de notre jeu de données nous avons utilisé Penthao en tant qu’ETL.

    Nous nous sommes heurtés à un problème sur les données de la CEREMA, chaque fichier était lié à un seul département or notre but était de crée un outil sur toute la France. Nous avons donc choisi de crée un script python qui serait appeler dans Pentaho qui fusionnerais les fichiers pour n’en avoir qu’un seul en entrer de nos données.

    Dans notre script Python on commence par définir le chemin vers le dossier ou sont placé tous les fichier Excel à empilé, ensuite on fait une boucle pour concaténer chacun de nos fichiers.

    Une fois notre fichier concaténer, pour des raisons d’optimisation on à choisis de supprimé les variables non utiles pour ne pas avoir à les importer dans Pentaho.

    Certaine des ventes présente dans le fichier concerne plusieurs propriétés et son donc sur plusieurs codes postaux. Nous avons donc également fait un traitement pour avoir une ligne égale à une transaction pour un biens en séparant tous les codes postaux.

    Ce Scripte Python est appelé à chaque fois que l’ont lancé notre flux et s’exécute donc avant la jointure avec les données de l’INSEE.

    4 - Résultats

    A la fin de l’execution du flux nous avons donc un seul fichier avec tous les éléments dont nous avions besoins. Ce fichier à donc été intégrer à notre Power BI afin de réaliser quelques visualisations sur les données.

    Sur ces cartes on peut voir le prix moyen des bâtiments suivant les communes a gauche et le prix moyen des terrains à droite.

    Dans notre visualisation nous avons ajouté des filtres, notamment pour choisir une période donnée dans le temps mais aussi pour filtrer le type de biens, uniquement les maison, les appartements ou les terrains par exemples.

    Ce projet a donc mêlé plusieurs de nos compétences, python pour la partie automatisation des données, Pentaho pour les liens et la structure des données mais aussi Power Bi pour la visualisation des données.
